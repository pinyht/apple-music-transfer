# Apple Music Transfer

Apple Music Transfer 是一个用于 Apple Music 数据备份与迁移的 Tampermonkey 用户脚本。

项目地址：

- GitHub: https://github.com/pinyht/apple-music-transfer
- Greasy Fork: https://greasyfork.org/zh-CN/scripts/568952-apple-music-transfer

它主要用于：

- 备份 Apple Music 资料库中的专辑、歌曲、播放列表
- 在不同 Apple Music 账号之间迁移数据
- 在不同区服之间迁移数据
- 重建资料库与播放列表
- 在 Apple Music 订阅中断、资料库和播放列表丢失后进行恢复

## 当前支持

- 导出专辑
- 导出歌曲
- 导出播放列表
- 导入专辑
- 导入歌曲
- 导入播放列表

## 适用场景

- 更换 Apple ID 后迁移资料库
- 从一个区服迁移到另一个区服
- 先导出做本地备份，再按需重新导入
- 播放列表重建
- Apple Music 订阅到期后未续费，后续重新订阅时资料库或播放列表丢失
- 不想为了保留资料库而长期维持订阅，先做本地备份，后续再恢复

## 为什么需要备份

Apple Music 的资料库和播放列表并不适合被当作永久存档。

在实际使用里，下面这些情况都可能导致你需要重新恢复数据：

- 更换账号
- 切换区服
- 长时间停用订阅
- Apple Music 订阅到期后资料库与播放列表丢失

这个工具的一个重要价值就是：

- 先把专辑、歌曲、播放列表导出成可保存的本地文件
- 即使之后不持续订阅，也能保留一份自己的备份
- 后续重新订阅时，可以再把资料库和播放列表恢复回来

## 安装方式

需要：

- 已安装 Tampermonkey
- 可正常访问 `music.apple.com`
- 已登录 Apple Music 账号
- 如果需要跨区服迁移资料，需要具备对应区服可用的网络代理环境

安装后，在 `music.apple.com` 页面中启用脚本即可使用。

建议通过脚本库搜索 `Apple Music Transfer` 后直接安装。

也可以直接通过下方地址安装：

- Greasy Fork 安装页： https://greasyfork.org/zh-CN/scripts/568952-apple-music-transfer

补充说明：

- 在直连网络环境下，Apple Music 网页版通常会被强制落到中国大陆区
- 因此如果要切换到其他区服进行导出或导入，通常需要先准备对应区服可用的代理网络环境

## 使用说明

### 1. 导出专辑 / 导出歌曲

- 打开 Apple Music 网页版
- 打开脚本面板
- 选择 `导出`
- 进入对应导出项
- 点击开始导出

脚本会自动抓取数据并导出：

- `csv`
- `json`

导出的 `json` 可用于后续导入。

### 2. 导出播放列表

- 进入某个播放列表详情页
- 打开脚本面板
- 选择 `导出 -> 导出播放列表`
- 点击开始导出

播放列表导出会保留：

- 播放列表名称
- 播放列表简介
- 歌曲顺序
- 歌曲 ID / 歌曲名 / 歌手名 / 专辑名

### 3. 导入专辑 / 导入歌曲 / 导入播放列表

- 打开脚本面板
- 选择 `导入`
- 进入对应导入项
- 点击 `导入 JSON`
- 再点击 `开始导入`

说明：

- 导入专辑、歌曲时，脚本会根据 ID 打开详情页并自动判断是否可加入资料库
- 导入播放列表时，脚本会根据歌曲 ID 逐条加入目标播放列表
- 如果目标播放列表不存在，脚本会自动新建
- 专辑迁移是按“整张专辑”恢复的。也就是说，如果你原来只收藏了某张专辑中的部分歌曲，使用专辑导入后，Apple Music 会把该专辑的全部歌曲加入资料库
- 如果你希望更精确地迁移到歌曲粒度，例如只恢复专辑中的部分歌曲，请使用“导出歌曲”和“导入歌曲”功能

## 注意事项

- 不同区服的版权与上架情况不同，部分歌曲或专辑可能无法导入
- 导入条目很多时，执行时间会比较长
- 播放列表导入依赖网页端菜单交互，执行期间不要手动干扰页面
- 建议先导出 `json` 做备份，再执行导入
- 即使是同一个账号、同一个区服，重新订阅后也可能出现资料库或播放列表丢失，因此建议定期备份
- 如果没有代理网络环境，跨区服切换可能失败，Apple Music 也可能自动回到中国大陆区
- 专辑导入恢复的是整张专辑，不会保留“只收藏专辑内其中几首歌”的粒度；如果需要按歌曲精确恢复，请使用歌曲导出与歌曲导入
